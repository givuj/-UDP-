#一
        1 主要学会了UDP是以包的形式传播
        2 程序在等数据的时候会进入阻塞状态recvfrom(1024)，让CPU先处理其他进程或者线程(目的是减少cpu占用消耗)，从而也导致程序无法执行系统异常命令（比如ctrl+c 停止程序）
        3 数据包来了会先放入内存中，然后操作系统告诉CPU数据来了，可以去程序中处理数据了
        3 一次性从内存拿也是有大小限制的，比如 recvfrom(1024)就是一次性拿1024字节，如果UDP数据包大于这个，大的会被舍弃，通过while ture重复提取数据包
        4 因为在阻塞的时候无法结束进程，除非等待时间超时，让操作系统把cpu还给程序，所以用select来优化，select在阻塞的时候还可以监听Ctrl+c停止程序命令
          selet  1.支持同时监听多个 UDP 套接字（极强的可扩展性）2.select 让操作系统帮忙监控套接字，有数据了就提醒程序可以拿了—— 这也是select能实现 “非阻塞读取”，而传统的recvfrom(1024)加超时需要等等待时间结束后才能执行接下来的操作
